# ADR-0005 — Минимизация зависимостей

Автор: Проект

Связанные документы:

---

## Контекст

Проект должен оставаться простым, быстрым и сопровождаемым одним разработчиком.

Каждая внешняя библиотека увеличивает долгосрочную стоимость поддержки: обновления, совместимость, безопасность, размер сборки и сложность понимания проекта.

Для текущего сайта большинство задач можно решать средствами платформы: Next.js, React, TypeScript, Tailwind CSS, HTML и CSS.

---

## Рассматриваемые варианты

- Добавлять библиотеки по мере удобства.
- Использовать UI-фреймворк.
- Минимизировать зависимости и добавлять их только при необходимости.

---

## Принятое решение

Проект придерживается принципа минимизации зависимостей.

Каждая новая библиотека должна иметь понятное обоснование.

Предпочтение отдаётся возможностям платформы.

Тяжёлые зависимости не используются без объективной необходимости.

---

## Причины

Решение выбрано для того, чтобы:

- сохранять простоту проекта;
- уменьшать размер клиентского кода;
- снижать риски устаревания;
- облегчать сопровождение одним разработчиком;
- избегать зависимостей ради модных или необязательных решений.

---

## Последствия

Проект остаётся легче для понимания и поддержки.

Новые зависимости требуют предварительного обоснования.

Ограничение решения состоит в том, что некоторые задачи могут требовать немного больше ручной реализации, если подключение библиотеки не оправдано.

---

## Что потребуется при пересмотре

Решение может быть уточнено, если появится задача, которую невозможно разумно решить средствами текущего стека, либо если библиотека существенно снизит сложность без ухудшения поддержки проекта.
